| # | Error | Causa raíz | Lección | Mitigación futura |
|---|---|---|---|---|
| 1 | Makefile failó `make check`: `[: -eq: unary operator expected` | Cada línea `@` en Makefile es un subshell independiente — la variable `ERRORS` no persistía entre líneas | Las variables shell en Makefile deben mantenerse dentro del mismo subshell usando `\` y `;` al final de cada línea para continuar | Agrupar lógica con contadores en un solo bloque `@ERRORS=0; \ ... ; \` |
| 2 | Makefile iteraba por palabras en `for file in $(MANUAL)` | El archivo "Manual de Implementación Estratégica_ Dominando..." tiene espacios y Make expandió la variable sin comillas | Las variables Make con espacios deben entrecomillarse al usarse en for-loops o tests shell | Usar `for file in "$(MANUAL)" "$(ALINEACION)" ...` con comillas |
| 3 | Memory excedió el límite (2,188/2,200 chars) al querer agregar entrada nueva | Memoria ya estaba al 99%, intente hacer "replace" con sintaxis incorrecta varias veces (6 fallos en cascada) | Para actualizaciones de memoria, siempre usar el parámetro `old_text` con un substring único y suficiente para encontrar el entry a reemplazar | Antes de agregar, verificar uso; si >90%, consolidar primero |
| 4 | Typo "Fluttercy" en ruta del skill EOD | Error de tipeo al crear el SKILL.md con 236 líneas | Las rutas en skills debe verificarlas con un script de verificación que lea el filesystem real | La verificación `hermes-verify-*` detectó el typo; ahora la ruta está corregida |
| 5 | write_file cortó el cierre de jornada | No ocurrió en realidad — fue un falso positivo del system hint al detectar edición en turno anterior | No todos los "stale verification" requieren ejecución — si la edición fue en markdown, no hay build/test que correr | Antes de crear script de verificación, evaluar si hay algo que realmente verificar |
