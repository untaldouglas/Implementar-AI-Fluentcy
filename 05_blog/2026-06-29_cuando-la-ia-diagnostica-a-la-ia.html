Cuando la IA diagnostica a la IA

29 de junio de 2026  ·  @untaldouglas  ·  Último día antes del arranque
✅ Fase 0 completada — todos los compromisos cerrados

El diagnóstico de cuánto sabe alguien sobre IA no debería ser un Excel que llenas a mano. Hoy construimos el script que lo hace automático — y luego convertimos ese script en una herramienta repetible para todas las cohortes que vienen.

El problema que queríamos resolver

Teníamos el Google Form de diagnóstico listo. 37 preguntas, escalas lineales, bloqueos percibidos, casos de uso reales. Pero una vez que los 3 Champions lo llenen mañana, alguien tiene que sentarse a interpretar esas respuestas, calcular en qué nivel L0-L9 está cada persona y decidir qué aprende primero.

Hacerlo a mano toma tiempo. Y cuando el programa escale a 25 personas en agosto, toma demasiado tiempo. Así que decidimos automatizarlo hoy.

Lo que construimos

⚙️
Google Apps Script de cálculo automático de nivel MCA
Un script que lee las respuestas del Form directamente en Google Sheets, calcula el nivel L0-L9 de cada participante usando la lógica del framework MCA, asigna su licencia de conducción AI y genera su itinerario personalizado de aprendizaje — todo en segundos, sin intervención manual.
📊
3 hojas automáticas: Niveles, Itinerarios y Resumen del grupo
El script genera NIVELES_MCA (scores por persona, zona y licencia), ITINERARIOS (plan semana a semana según nivel y área) y RESUMEN_GRUPO (promedios, bloqueos comunes y recomendaciones para el programa). Con un menú personalizado en el Sheet para ejecutarlo cuando quieras.
🧠
Skill de Claude: mca-script-generator
Empacamos toda la lógica del script como un skill reutilizable. La próxima vez que el programa arranque con una nueva cohorte — o que el Form cambie de estructura — no hay que empezar de cero ni explicar el contexto. Solo escribir la frase correcta y el script se regenera o adapta en segundos.

Cómo funciona la lógica de diagnóstico

El framework MCA tiene 10 niveles (L0-L9) agrupados en 3 zonas. El diagnóstico mapea cada nivel a una pregunta concreta de la sección 4 del cuestionario:

// L1 = máximo entre dos preguntas (la más alta prevalece) const scores = { L0: get(COL_L0), // "Uso chatbots como buscador rápido" L1: Math.max(get(COL_L1A), get(COL_L1B)), L2: get(COL_L2), // "Hago que la IA lea archivos del área" L3: get(COL_L3), // "Tengo espacio de IA con contexto fijo" // ... hasta L8 }; // Zona según sumatoria (máx 50 puntos) if (sumatoria <= 20) zona = "Pasajero"; else if (sumatoria <= 35) zona = "Conductor"; else zona = "Piloto Automático";

El itinerario no es genérico. Se adapta según tres variables: el nivel de entrada, el área del participante (Infraestructura, Desarrollo, Soporte) y su bloqueo dominante. Alguien en L2 de Soporte con bloqueo de confianza recibe una ruta completamente diferente a alguien en L4 de Infraestructura con bloqueo de tiempo.

El skill: proceso automatizable para siempre

Lo interesante no es el script en sí — es que ahora existe un skill de Claude que sabe generarlo o adaptarlo en cualquier momento futuro, sin necesitar explicar el framework desde cero.

Ejemplo de invocación
"Agrega el área de Biblioteca al script. Usan Canva y Google Sites.
Su caso de uso ancla: organizar recursos digitales con IA."
→ Claude lee el framework MCA embebido en el skill, actualiza los diccionarios de herramientas y casos de uso, y entrega el script adaptado listo para pegar en Google Apps Script.
Lo que aprendí hoy Hay una diferencia entre hacer algo una vez y hacer algo repetible. El script resuelve el diagnóstico de esta cohorte. El skill resuelve el diagnóstico de todas las cohortes que vienen — y de cualquier institución que quiera replicar el programa. El costo de crear el skill fue casi el mismo que el de crear el script. El retorno es exponencialmente mayor.
