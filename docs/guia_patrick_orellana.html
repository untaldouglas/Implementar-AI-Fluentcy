Día 1 · Lección 1
¿Qué es un prompt para infraestructura?
30 min · Hermes Agent en tu estación de trabajo
⚠️
Nota de inicio: Tu setup de Hermes está pendiente por el incidente del 01/07. Completa primero la instalación con Douglas, luego empieza esta guía. Fecha límite de setup: 02/07/2026.
Concepto clave
Un prompt para infraestructura es como escribir un procedimiento técnico: specificas el sistema, el entorno y lo que necesitas. Cuanto más específico eres con el contexto técnico, más preciso es el resultado — igual que con la documentación de sistemas.

Iniciar Hermes para infraestructura

# Para generar scripts Bash/PowerShell usa el toolset terminal
hermes chat -t terminal -q "genera un script para..."

# Para análisis y documentación (sin ejecutar comandos)
hermes chat -q "explica / genera / redacta..."

# Sesión interactiva
hermes

Práctica — compara estos prompts

Prompt A — Sin contexto:

hermes chat -q "genera un script para monitorear servidores"

Prompt B — Con contexto de infraestructura:

hermes chat -t terminal -q "Eres un administrador de infraestructura universitaria. Genera un script Bash que verifique el estado de conectividad de los servidores críticos de la red. El script debe: hacer ping a una lista de IPs (usar IPs ficticias de ejemplo), mostrar cuáles responden y cuáles no, registrar el resultado con timestamp en un archivo de log, y enviar alerta por correo si algún servidor no responde. Usa datos ficticios para IPs y correos."

  • Ejecuté el Prompt B y noté la diferencia de calidad

Día 2 · Lección 1 (práctica)
3 prompts de infraestructura para tu flujo real
45 min · Hermes Agent

Prompt 1 — Script de verificación de respaldos

hermes chat -t terminal -q "Genera un script Bash que verifique si los respaldos del día anterior fueron completados exitosamente. El script debe: verificar la existencia del archivo de respaldo con fecha de ayer, validar que el tamaño sea mayor a un mínimo configurable, registrar el resultado en un log con timestamp, y retornar exit code 0 si está bien o 1 si hay problema. Usa rutas ficticias como /backup/ejemplo/."

Prompt 2 — Reporte de estado de infraestructura

hermes chat -q "Genera una plantilla de reporte mensual de estado de infraestructura universitaria. Secciones: resumen ejecutivo, disponibilidad de servicios críticos (%), incidentes del mes, respaldos (estado y tamaño), capacidad de almacenamiento, acciones pendientes. Formato: tabla + narrativa breve por sección."

Prompt 3 — Procedimiento de respuesta a incidente

hermes chat -q "Redacta un procedimiento paso a paso para responder a un incidente de caída de servidor en una universidad. Incluye: detección, escalamiento, comunicación al equipo, acciones de recuperación, verificación del servicio, y documentación post-incidente. Formato: numerado con checkboxes."
🔒
Tu bloqueo principal — Delegación segura: Hermes (y el modelo LLM detrás) procesa lo que le envías. Nunca incluyas: IPs reales de producción, credenciales, tokens, contraseñas, nombres de servidores internos reales, ni configuraciones de red específicas. Usa siempre datos ficticios o genéricos. La lógica del script sí es segura de compartir.

  • Verifiqué que ningún prompt contiene datos reales de producción

Día 3 · Lección 2
Refinamiento iterativo de scripts
35 min · Hermes Agent modo interactivo
Concepto clave
Un script generado por IA es un borrador de inicio, no el producto final. El valor está en el ciclo de refinamiento: pides, revisas, corriges, vuelves a pedir. Igual que con un junior técnico al que vas guiando.

Ciclo de refinamiento de script en Hermes

# Inicia sesión interactiva
hermes

# Toma el script del Día 2 (ej: verificación de respaldos) y refina:
> [pega el script generado el Día 2]
> Agrega manejo de errores — que el script no falle silenciosamente.
> Agrega un parámetro opcional para especificar la fecha a verificar.
> Convierte el script de Bash a PowerShell manteniendo la misma lógica.
> Agrega comentarios explicativos en español a cada sección.
> /quit

Actividades del Día 3

  • Inicié Hermes en modo interactivo
  • Pedí manejo de errores al script del Día 2
  • Pedí la versión PowerShell del mismo script
  • Pedí comentarios explicativos en español
  • Guardé la versión final refinada (Bash + PowerShell)
💡
Sobre conocimiento técnico (tu bloqueo): Hermes puede no conocer las particularidades exactas de tu infraestructura. Es normal. Tú pones el contexto específico, Hermes pone la estructura y el código base. La combinación es más potente que cualquiera de los dos solos.

Día 4 · Lección 2 + Seguridad
SOUL.md + protocolo de delegación segura
35 min · Hermes Agent + editor de texto
Dos cosas hoy
1. Crear el SOUL.md de Infraestructura para que Hermes siempre tenga contexto de tu área.
2. Definir tu protocolo personal de qué SÍ y qué NO delegar a la IA.

Tu SOUL.md de Infraestructura

# SOUL — Agente de Infraestructura UJMD

Eres un asistente de operaciones de infraestructura de la Dirección
de Servicios Informáticos de la UJMD, El Salvador.

## Tu especialidad
- Scripts Bash y PowerShell para administración de sistemas
- Verificación de respaldos y monitoreo de servicios
- Generación de reportes técnicos y procedimientos
- Documentación de infraestructura

## Reglas de seguridad (críticas)
- NUNCA uses IPs reales, hostnames reales ni credenciales
- Usa siempre IPs ficticias (ej: 192.168.x.x) en los ejemplos
- Si ves credenciales en lo que te comparto, advierte de inmediato
- Los datos corporativos NO se comparten con modelos LLM

## Tono
Técnico, preciso, con comentarios explicativos en español.

Tu protocolo de delegación segura (llena esto)

Define tus reglas personales
✅ SÍ puedo delegar a Hermes:
— Generar scripts con lógica (usando datos ficticios)
— Plantillas de reportes y procedimientos
— Documentación técnica
— Análisis de logs (con datos anonimizados)

❌ NUNCA delego a Hermes:
— IPs reales de producción
— Credenciales, tokens, contraseñas
— Configuraciones de firewall reales
— Datos de usuarios o estudiantes
— [agrega los tuyos según tu contexto]

60 min · Hermes Agent (con SOUL.md activo)

Tu caso ancla: Generación de scripts de monitoreo y verificación de respaldos — las tareas que mencionaste como candidatas para automatizar.

Script completo: verificación diaria de infraestructura

hermes chat -t terminal -q "Genera un script Bash completo de verificación diaria de infraestructura universitaria. El script debe:
1. Verificar conectividad con ping a lista de servidores ficticios (web, BD, respaldos)
2. Verificar que el respaldo de ayer existe y su tamaño supera 1GB
3. Verificar espacio en disco (alerta si supera 80% de uso)
4. Generar un reporte resumido en /var/log/infra_check_YYYY-MM-DD.log
5. Enviar email de alerta si alguna verificación falla (destinatario ficticio)
Todo con datos ficticios. Incluir manejo de errores y comentarios en español."

Paso 2 — Adaptar a tu entorno real (sin datos sensibles)

Una vez obtengas el script, ajusta manualmente:

  • Lista de servidores a verificar (puedes usar nombres genéricos como servidor-web, servidor-bd)
  • Ruta real de respaldos (sin credenciales)
  • Umbrales de disco según tu infraestructura real

Actividades del Día 5

  • Generé el script completo con Hermes (con SOUL.md activo)
  • Revisé el script línea a línea — entiendo qué hace cada parte
  • Identifiqué qué ajustes necesita para mi entorno real
  • Probé el script en un entorno de prueba (no producción)
  • Guardé la versión final con mis ajustes
⚠️
Antes de usar en producción: Prueba siempre en un entorno de desarrollo o con datos ficticios. Un script mal configurado puede interrumpir servicios. Valida con Douglas antes de poner en producción.
