Lección 1 · L2 → L3

Instalación del Core Agent

Tiempo estimado
10–15 minutos
Nivel
L2 → L3
Objetivo
Tener hermes funcionando en tu terminal

Prerrequisitos

RequisitoDetalle
Sistema operativomacOS, Linux, o Windows (incluye WSL)
TerminalTerminal.app, PowerShell, o bash
PythonNo obligatorio — el instalador lo gestiona
InternetConexión estable (~50 MB de descarga)
⚠️
Red UJMD: La red institucional puede bloquear el certificado SSL de npm. Si la instalación falla, usa una red externa (hotspot móvil) e intenta de nuevo.

Paso 1 — Instalar Hermes Agent

Abre tu terminal y ejecuta una de estas opciones:

Opción A — Instalador oficial (recomendada)

curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

Este script instala automáticamente: uv, Python (si no está), el entorno virtual de Hermes, y el comando hermes en tu PATH.

Opción B — vía pip (si ya tienes Python 3.11+)

pip install hermes-agent

Paso 2 — Verificar la instalación

hermes doctor

Resultado esperado:

✓ Python version: 3.11.x
✓ Virtual environment: active
✓ Hermes Agent: installed
⚠ API key: not configured (lo haremos en la Lección 2)
💡
Si hermes doctor reporta errores, ejecuta hermes doctor --fix para intentar autocorrección.

Paso 3 — Primer contacto

hermes

Verás el banner de Hermes y el prompt de chat. Escribe un mensaje de prueba:

> Hola, soy [tu nombre] y estoy configurando Hermes Agent para el piloto AI Fluency.

Para salir: escribe /quit

Checklist — Lección 1

  • □
    hermes responde en la terminal
  • □
    hermes doctor ejecuta sin errores críticos
  • □
    Pude iniciar una sesión de chat (aunque falle por API key)
  • □
    Sé cómo salir con /quit

Problemas comunes

ProblemaSolución
command not found: hermesCierra y reabre la terminal. Si persiste: source ~/.bashrc (Linux) o reinicia Terminal.app (macOS)
Error de permisos en macOSsudo chown -R $(whoami) ~/.hermes
Python no encontradoEl instalador maneja esto; si usaste pip, instala Python 3.11+ primero
Windows: curl no existeUsa WSL o descarga el script manualmente
Lección 2 · L3 → L4

Configuración del Modelo y Provider

Tiempo estimado
15–20 minutos
Nivel
L3 → L4
Objetivo
Conectar Hermes con Claude + Telegram

Prerrequisitos

  • ✓ Lección 1 completada (hermes doctor pasa sin errores críticos)
  • ✓ Clave API de Anthropic (proporcionada por Douglas)
  • ✓ Tu cuenta de Telegram activa

Paso 1 — Asistente de setup interactivo

hermes setup

El wizard te guiará. Selecciona:

SecciónValor a seleccionar
Model → Provideranthropic
Model → Modelclaude-sonnet-4 (recomendado)
Model → API KeyPega la clave que te dio Douglas
Terminal → Backendlocal (valor por defecto)
ToolsHabilita: web, terminal, file, skills

Paso 2 — Alternativa: configuración manual

Seleccionar modelo directamente

# Menú interactivo para seleccionar provider + model
hermes model

# Ver ruta del archivo .env
hermes config env-path

Añade esta línea al archivo .env:

ANTHROPIC_API_KEY=sk-ant-XXXXX-tu-clave-aqui
🔒
IMPORTANTE: Nunca compartas tu API key por correo, chat, ni screenshots. Si necesitas transmitirla, usa canales seguros.

Paso 3 — Verificar que el modelo responde

hermes chat -q "¿Cuál es la capital de El Salvador?"

Resultado esperado: el agente responde "San Salvador" en pocos segundos. Si responde correctamente, el modelo está configurado ✅

Paso 4 — Gateway de Telegram (recomendado)

El gateway te permite interactuar con Hermes desde Telegram.

# Iniciar setup del gateway
hermes gateway setup

# Selecciona "telegram" y sigue las instrucciones:
# 1. Crea un bot en Telegram hablando con @BotFather
# 2. Copia el token del bot
# 3. Pégalo cuando el setup lo solicite

# Iniciar el gateway
hermes gateway start

# Verificar estado
hermes gateway status
# Resultado esperado: telegram: connected ✓
💡
Si prefieres esperar a la Sesión 0 para configurar Telegram, puedes saltar este paso. Es opcional para el setup inicial.

Checklist — Lección 2

  • □

Tiempo estimado
10 minutos

Paso 1 — Diagnóstico completo

hermes doctor --fix

✓ Python version: 3.11.x
✓ Virtual environment: active
✓ Hermes Agent: installed
✓ API key: configured (anthropic)
✓ Model: claude-sonnet-4
✓ Terminal: local backend
✓ Tools: web, terminal, file, skills
⚠️
Cualquier línea con ✗ o ⚠ debe resolverse antes de Sesión 0. Contacta a Douglas si no puedes resolverla.

Paso 2 — Prueba de herramientas (toolsets)

Prueba A — Web search

hermes chat -q "Busca en la web: Universidad Dr. José Matías Delgado El Salvador"

Verifica que el agente use web_search y devuelva información real sobre la UJMD.

Prueba B — File operations

hermes chat -q "Crea un archivo llamado prueba_champion.txt en mi directorio home con el texto: Setup de Hermes Agent completado por [tu nombre]"

# Verificar que fue creado:
cat ~/prueba_champion.txt

Prueba C — Skills

# Listar skills disponibles
hermes skills list

# Cargar un skill y consultarlo
hermes chat -s hermes-agent -q "¿Qué puedes hacer según tu documentación?"

Paso 3 — Status global

hermes status --all

Copia y guarda este output — lo compartirás con Douglas como evidencia de setup completado.

Paso 4 — Limpieza

rm ~/prueba_champion.txt

Cheat Sheet — Comandos Esenciales

Imprime o ten abierta esta hoja durante el setup y el uso diario.

🚀 Setup inicial (una sola vez)

# Instalar Hermes Agent
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

# Verificar instalación
hermes doctor

# Configurar modelo y provider (interactivo)
hermes setup

# O seleccionar modelo directamente
hermes model

# Ver config actual
hermes config

💬 Chat

# Chat interactivo
hermes

# Consulta única
hermes chat -q "tu pregunta aquí"

# Con un skill cargado
hermes chat -s hermes-agent -q "¿qué puedes hacer?"

# Con toolsets específicos
hermes chat -t web,terminal -q "busca X y crea un archivo"

🔧 Configuración

# Ver config completa
hermes config

# Editar config.yaml
hermes config edit

# Setear un valor
hermes config set section.key value

# Ver ruta del .env
hermes config env-path

# Verificar estado global
hermes status --all

🛠 Tools y Skills

# Listar toolsets
hermes tools list

# Habilitar/deshabilitar (interactivo)
hermes tools

# Listar skills instalados
hermes skills list

# Buscar skills
hermes skills search "keyword"

# Inspeccionar un skill
hermes skills inspect ID

📱 Gateway (Telegram)

# Setup
hermes gateway setup

# Iniciar
hermes gateway start

# Estado
hermes gateway status

# Reiniciar / Detener
hermes gateway restart
hermes gateway stop

📅 Cron Jobs

# Listar
hermes cron list

# Crear (ejemplo: lunes-viernes 9am)
hermes cron create "0 9 * * 1-5"

# Pausar / Reanudar
hermes cron pause ID
hermes cron resume ID

💻 Sesiones

# Listar sesiones recientes
hermes sessions list

# Reanudar última sesión
hermes --continue

# Reanudar por ID
hermes --resume SESSION_ID

⌨️ Comandos dentro del chat (slash commands)

ComandoAcción
/helpVer todos los comandos
/newNueva sesión limpia
/modelCambiar modelo
/toolsGestionar herramientas
/skill <nombre>Cargar un skill
/skillsBuscar/instalar skills
/statusInfo de la sesión
/usageUso de tokens
/quitSalir

🆘 Emergency

# Algo no funciona
hermes doctor --fix

# Reset completo de config
hermes setup

# Ver logs del gateway
cat ~/.hermes/logs/gateway.log

Referencia de términos

Glosario

Términos clave de Hermes Agent y del piloto AI Fluency. Mantén esta referencia a mano durante el programa.

Términos técnicos esenciales

TérminoDefinición
AgentEl asistente de IA que ejecuta tareas usando herramientas. En nuestro caso, Hermes Agent.
ProviderEl servicio que provee el modelo de IA. Ej: Anthropic (Claude), OpenAI, OpenRouter.
ModelEl modelo de lenguaje específico. Ej: claude-sonnet-4, gpt-4o.
ToolsetConjunto de herramientas que el agente puede usar: web, terminal, file, skills, etc.
SkillProcedimiento reusable guardado como documento Markdown. Carga conocimiento especializado al agente.
GatewayServicio que conecta Hermes con plataformas de mensajería (Telegram, Discord, Slack).
ProfileInstancia independiente de Hermes con su propia config, skills, memoria y sesiones.
SessionUna conversación con Hermes. Se puede resumir, exportar y reanudar.
SOUL.mdArchivo de identidad del agente. Define quién es y cómo se comporta.
CLICommand Line Interface — la terminal donde escribes comandos hermes ...
TUIText User Interface — la interfaz visual en terminal (modo interactivo).

Comandos clave

ComandoQué hace
hermesInicia chat interactivo
hermes chat -q "..."Consulta única (non-interactive)
hermes setupAsistente de configuración
hermes doctorDiagnóstico del sistema
hermes modelSeleccionar modelo/provider
hermes toolsHabilitar/deshabilitar toolsets
hermes skills listListar skills instalados
hermes gateway startIniciar gateway de mensajería
hermes status --allEstado global de todos los componentes
/skill <nombre>Cargar un skill en la sesión actual
/quitSalir del chat
